# PikPak Assistant

PikPak 网页端批量处理助手，专门用来处理番号重命名、小文件清理和子文件夹拆分。

## 现在这版包含什么

- 扫描番号后批量重命名
- 重命名前先预览，再确认执行
- 文件名格式支持把关键词放在番号后面
- 最多保留前 `3` 个关键词
- 独立清理小于 `100MB` 的普通文件
- 选中文件夹后递归扫描所有层级的小文件
- 独立拆分子文件夹
- 把所有层级子文件夹里的文件直接搬到最外层母文件夹
- 空子文件夹自动删除
- 遇到同名冲突自动跳过，不强行覆盖
- 删除空文件夹前会额外重试 `2-3` 次

## 重命名亮点

- 支持常规番号识别
- 支持从页面获取标题后重命名
- 支持从 `av-wiki` 标签提取关键词
- 支持从标题正文继续补关键词
- 支持从文件名继续补关键词
- 支持激进型标题补词兜底
- 关键词位置是：

```text
番号 [关键词][关键词][关键词] 标题.ext
```

示例：

```text
ABP-123 [人妻][NTR][巨乳] 标题.mp4
```

## 安装

1. 打开 [PikPak Assistant.user.js](./PikPak%20Assistant.user.js)
2. 用油猴 / Tampermonkey 安装
3. 打开 `mypikpak.com`、`mypikpak.net` 或 `pikpak.me`

## 使用

### 1. 扫描番号并重命名

1. 选中文件或文件夹
2. 点击 `扫描番号`
3. 等待扫描完成
4. 在预览里确认新文件名
5. 点击 `确认重命名`

### 2. 清理小文件

1. 选中文件夹
2. 点击 `清理小文件`
3. 脚本会递归扫描所有层级
4. 找出所有小于 `100MB` 的普通文件
5. 确认后移入回收站

### 3. 拆分子文件夹

1. 选中文件夹
2. 点击 `拆分子文件夹`
3. 脚本会递归扫描所有层级子文件夹
4. 把子文件夹里的文件全部搬到最外层母文件夹
5. 空子文件夹自动删除
6. 同名冲突文件自动跳过

## 配置

- `在文件名开头增加发行日期`
- `修复文件扩展名`
- 排序方式
- 排序方向

## 注意

- 删除和移动都会真实执行
- 小文件清理是独立功能，不依赖扫描番号
- 拆分子文件夹是独立功能，不依赖扫描番号
- 遇到冲突文件会跳过，不会覆盖原文件
- 如果没有识别到番号，通常是文件名规则不匹配
- 如果标题里没有直接标签，脚本会继续尝试从标题正文补关键词
